# Roadmap

## Phase 1 - Backend MVP

- Run Docker-based mini-apps through a structured API
- Validate app name, image, and port configuration
- Read container logs
- Expose health and status endpoints
- Keep command execution controlled and security-aware

## Phase 2 - Local Dashboard

- Show installed mini-apps
- Start / stop / restart apps
- Display logs and container status
- Provide a simple setup wizard for non-technical users

## Phase 3 - App Catalog

- Add predefined mini-app templates
- Support local business tools such as dashboards, forms, mini CRM, and reporting tools
- Add app metadata and configuration presets
- Make installation feel closer to choosing an app than running infrastructure manually

## Phase 4 - Safer Runtime

- Add app allowlist
- Add permission model
- Improve environment variable handling
- Add better error handling and update flow
- Add clearer separation between public templates and private runtime logic
